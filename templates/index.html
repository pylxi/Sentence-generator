Under the hood
How distractors are generated
1
Layer 1 — POS match (always applied)
Distractors must share the same part of speech as the target word. A noun distractor for a verb gap is grammatically impossible — learners can reject it without any vocabulary knowledge.
2
Layer 2 — CEFR proximity (±1 band)
Only words from the target band and its immediate neighbours (e.g. A2+B1+B2 for a B1 word) are used. A1 words are too easy to eliminate; C1+ words are unfairly hard.
3
Layer 3 — Three semantic types
semanticWordNet siblings (co-hyponyms sharing the same hypernym) or near-synonyms. These occupy the same semantic category as the target — the hardest type to eliminate without genuine word knowledge. Source: Susanti et al. (2018).
formLevenshtein distance ≤ 3 from the target. Tests orthographic discrimination — especially relevant for Japanese EFL learners.
unrelatedRandom same-POS word from a different topic. Gives learners a clear wrong answer if they simply don't know the target.
4
Sentence-frame plausibility filter
A lightweight suffix check removes distractors that morphologically can't fit the inflection in the sentence (e.g. comparative "-er" only accepted for short adjectives). No API call needed — runs instantly.
⚠ Fallback
If fewer than 3 valid distractors are found after all layers (common for rare POS like adverbs), the sentence is flagged. You can still approve what's available, or skip it — the fallback reason is shown on the card.
No API calls
Distractor generation is entirely local — it reads the CEFR-J wordlist CSVs and runs the selection logic in Python. Zero tokens spent.
